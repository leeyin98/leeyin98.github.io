第5讲 异常处理与程序调试

前言

第6讲主要讲述：异常处理；自定义异常；断言及程序的测试；程序的调试

正文

1. 异常处理

1.1 异常定义

异常，对应着Java运行错误的处理机制

基本写法
try{
    语句组;
}catch(Exception ex){
    异常处理语句组;
}
异常处理流程

1.抛出(throw)异常
2.运行时系统在调用栈中查找
3.捕获(catch)异常的代码
相关语句：

抛出异常：`throw 异常对象;`
捕获异常：
try{
    语句组
}catch(异常类名 异常形式参数名){
    异常处理语句组;
}catch(异常类名 异常形式参数名){
    异常处理语句组;
}finally{
    异常处理语句组;
}
其中，catch语句可以是0或多个，可以没有fanally语句

1.2 异常分类


Error：JVM的错误
Exception：异常，其构造方法为：

  • public Exception();
  • public Exception(String message);
  • Exception(String message,Throwable cause);

1.3 多异常处理

子类异常要写在父类异常前面
finally语句：无论是否有异常都要执行，即使其中有break,return等语句

1.4 受检的异常

Exception分为两种：

  • RuntimeException及其子类，可以不明确处理
  • 否则，称为受检的异常(checked Exception)
    受检的异常，要求明确进行语法处理：
  • 要么捕获(catch)
  • 要么抛出(throws)：在方法的签名后面用throws xxxx来声明

2. 自定义异常

2.1 创建用户自定义异常

继承自Exception类或某个子Exception类
定义属性和方法，或重载父类的方法

2.2 重抛异常及异常链接

对于异常，不仅要进行捕获处理，优势也要将此异常进一步传递给调用者，以便让调用者也能感受到这种异常，这时可以在catch语句块或finally语句块中采取以下三种方式：

  • 将当前捕获的异常再次抛出
    • throw e;
  • 重新生成一个异常并抛出，如
    • throw new Exception("some message");
  • 重新生成并抛出一个新异常，该异常中包含了当前异常的信息，如
    • throw new Exception("some message",e);可用e.getCause()得到内部异常

3. 断言及程序的测试

3.1 断言(assertion)

assert的写法：

  • assert 表达式;
  • assert 表达式：信息;
    例：
    class Assertion{
  public static void main(String[] args){
      assert hypotenuse(3,4)==5 : "算法不正确";   //assert写法
  }
  static double hypotenuse(double x,double y){
      return Math.sqrt(x*x+y*y+1);
  }
}
    调试程序时，若表达式部位true，则程序会产生异常并输出相关的错误信息

3.2 程序的测试及Junit

Eclipse中

  • 项目上右键-New-Junit Test Case
  • 项目上右键-Run as-Junit Test

3.3 测试函数

@Test用来标注测试函数
测试中常用的语句：

  • fail(信息); //表示程序出错
  • assertEquals(参数1,参数2); //表示程序要保证两个参数相等
  • assertNull(参数); //表示参数要为null

4. 程序的调试

4.1 程序中的错误

程序中的错误，通常分为三大类：

  • 语法错误(Syntax error)
    • 编辑、编译器发现
  • 运行错误(Runtime error)
    • 异常处理机制
  • 逻辑错误(Logic error)
    • 调试(debug)、单元测试(unit test)

4.2 程序的调试(debug)

步骤

1.在IDE中，项目上右键，debug as…
2.进入到调试试图(debug perspective)

调试的手段
  • 断点(breakpoint)：
    • 切换断点：鼠标右击编译器左边条；Eclipse快捷键：Ctrl+Shift+B
  • 跟踪(trace)：
    • Eclipse快捷键
    • 逐语句执行：F5
    • 逐过程执行：F6
    • 跳出函数：F7
    • 运行到光标处：Ctrl+R
  • 监视(watch)
    • 即时监视：鼠标指向变量
    • 快速监视：点右键，Inspector
    • 添加监视：点右键，Watch
